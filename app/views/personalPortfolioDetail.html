Container

A container is an element designed to contain page elements to a reasonable maximum width based on the size of a user's screen.

Grid

Grids divide horizontal space into indivisible units called "columns". All columns in a grid must specify their width as proportion of the total available row width. I chose an 16 column count to allow per row. And each website element span a certain number of columns.

A vertical rhythm was used to achieve visual consistency in website design.

Baseline Grid

No matter what font-size any text element is, its line-height is always an even multiple of a consistent unit of rhythm. I chose the base line-height as 20px, and base font-size as 14px. Then every website element's height, padding and margin is a multiple of base line height.

Yeoman was used to kickstart this project, prescribing best practices and tools to help me stay productive.

Yeoman Workflow

The Yeoman workflow comprises three types of tools for improving my productivity and satisfaction when building my website: the scaffolding tool (yo), the build tool (Gulp, Grunt etc) and the package manager (like npm and Bower).

Yo angular generator was used to scaffolds out my website, writing my Gulpfile build configuration and pulling in relevant build tasks as well as npm and bower package manager dependencies that I needed for my build.

Gulp was used to build and preview my website.

npm and bower was used for dependency management. npm focused on managing Node.js modules like Gulp and all the packages that used in Gulp build, while bower was mainly used for managing front-end libraries.

Leverage and customize front-end framework to develop fast and powerful web interfaces.

Using Sass to Set Up Baseline Scheme

Sass provides a great toolbox to generate consistent vertical rhythm and color scheme.

$base-font-size: 1rem; //html element font size is set as 14px
$base-line-height: 1.4285rem; //20px

/*Vertical rhythm example*/
button {
    height: 2 * $base-line-height;
}
$blue:      #00A9FF; //HSB 200,100,50
$white:     #F5F5FA; //HSB 244,2,98
$grey:      #706F7F; //HSB 244,2,50
$dark-grey: #383740; //HSB 244,2,25
$black:     #2D2C33; //HSB 244,2,20

/*Color scheme example*/
button {
    color: $blue;
}

Angular UI-Router

Angular UI-Router was used to update the browser's URL and manage the transition as the audience navigates through the app. It maps the URL to a specific "state" within my website. Obviously, the website has four main states - "About Me", "Projects", "Photos" and "Videos", and "Projects" has its sub-state of each project details.

Angular Services

Angular Services were used to store detail information of project, photo and video, which would be fetched by Angular Controller later to feed to web page.

/*Angular Services store data*/
angular.module('projectApp')
  .factory('photoFactory',function(){
    var photos = [{
      _id: 0;
      location: 'HaleakalƒÅ National Park, Maui, Hawaii';
      image: '/images/photos/IMG_9826.jpg';
      date: '2016.04';
    },
    //...
    ];

    var photoFac = {};
    photoFac.getPhotos() = function(){
      return photos;
    };

    return photoFac;
  });
Angular Controller

Angular Controllers were used to fetch data from Angular Services and feed the web page.

/*Angular Controllers fetch data*/
angular.module('projectApp')
  .controller('PhotoController', ['$scope','photoFactory',function($scope,photoFactory){
    var photos = photoFactory.getPhotos();
    $scope.photos = photos;
    //...
  }]);
/*Angular ng-repeat directive displays data in web page*/
<div ng-controller="PhotoController" ng-repeat="photo in photos">
  <img ng-src={{photo.image}} />
  <div>{{photo.location}}</div>
  <div>{{photo.date}}</div>
</div>
